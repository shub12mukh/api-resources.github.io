High resolution time API

The ECMAScript Language specification [ECMA-262] defines the Date object as a time value representing time in milliseconds since 01 January, 1970 UTC. For most purposes, this definition of time is sufficient as these values represent time to millisecond precision for any instant that is within approximately 285,616 years from 01 January, 1970 UTC. The DOMTimeStamp is defined similarly [WEBIDL]

In practice, these definitions of time are subject to both clock skew and adjustment of the system clock. The value of time may not always be monotonically increasing and subsequent values may either decrease or remain the same.

For certain tasks this definition of time may not be sufficient as it does not allow for sub-millisecond resolution and is subject to system clock skew. For example:

  1. When attempting to accurately measure the elapsed time of navigating to a Document, fetching of resources or execution of script, a monotonically increasing clock with sub-millisecond resolution is desired.
  2. When calculating the animation state from script, developers will need to accurately know the amount of time that has elapsed in the animation in order to properly update the next scene of the animation.

The DOMHighResTimeStamp type, performance.now method, and performance.timeOrigin attributes of the Performance interface resolve the above issues by providing monotonically increasing time values with sub-millisecond resolution.

Properties and methods

performance.now()

performance.now() returns the number of milliseconds, with microseconds in the fractional part, from performance.timing.navigationStart(), the start of navigation of the document, to the performance.now() call.

Network information API

The Network Information API provides access to the connection type the system is using to communicate with the network, cellular, Wi-Fi, Bluetooth, etc. It also provides a means for scripts to be notified if the connection type changes. This is to allow developers to make dynamic changes to the DOM and/or inform the user that the network connection type has changed.

Properties and methods

effectiveType property

The effectiveType read-only property of the NetworkInformation interface returns the effective type of the connection meaning one of 'slow-2g', '2g', '3g', or '4g'. This value is determined using a combination of recently observed, round-trip time and downlink values.

onchange event handler

The NetworkInformation.onchange event handler contains the code that is fired when connection information changes, and the change is received by the NetworkInformation object.

Vibration API

The window.navigator object has vibrate method which accept one parameter, i.e. the time in miliseconds

It is mainly used for making vibrational feedback.

The device must be compatible to run the vibrate method

Properties and methods

vibrate()

The vibrate() method of the navigator interface accepts one parameter, which can be an integer or an array of integers.

It is basically the duration of the vibration

Will be vibrated for a duration of 2 seconds using navigator.vibrate(2000)

Screen orientation API

The screen orientation api provided the ability to read the orientation of the viewport, detect change of the state.

Also, it is used to lock the orientation and unlock too.

Properties and methods

type property

The type property of the window.screen.orientation interface returns either of the four values of orientation.

  1. portrait-primary
  2. portrait-secondary
  3. landscape-primary
  4. landscape-secondary

lockOrientation()

The lockOrientation() method locks the device’s screen as if the device was physically rotated in a certain position.

The lockOrientation() method accepts one of the orientation state as a parameter

unlockOrientation()

The unlockOrientation() method is used to release a previously set lock and allows the screen to rotate again in every position.

onchange event

The onchange event that is fired every time the screen orientation changes.

Resize the viewport or increase and decrease the width and the height of the screen to see changes accordingly.

Also, check the console for the orientation onchange event

Speech Synthesis API

The Web Speech API provides provides both Speech recognition and Text to speech functionality.

Convert text to speech using SpeechSynthesisUtterance object

The SpeechSynthesisUtterance object represents the text that will be converted to speech by the synthesizer

Lets check out the properties provided by theSpeechSynthesisUtterance object

Properties and methods

text

text property of the SpeechSynthesisUtterance object is used to set set the text that will be converted to speech

lang

lang property of the SpeechSynthesisUtterance object is used for representing the language of the speech synthesis for the utterance (for example “en-GB” or “it-IT”).

volum

volume is a number representing the volume for the text. It ranges from 0 (minimum) to 1 (maximum) inclusive, and the default value is 1.

rate

rateis the number representing the speaking rate for the utterance. It is relative to the default rate for the voice.

pitch

pitchis the number representing the speaking pitch for the utterance. It ranges from 0 (minimum) to 2 (maximum) inclusive. The default value is 1.
